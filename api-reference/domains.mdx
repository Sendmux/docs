A **domain** is the fully qualified hostname that Sendmux sends mail from on your behalf — for example, `mail.acme.com`. Each team can register multiple domains, and every custom mailbox lives under one of them.

For the end-to-end walkthrough — adding records at your DNS provider, troubleshooting failed verifications, and deleting domains cleanly — see [Domain management](/guides/domain-management).

## Endpoints

| Endpoint                                    | Description                                                                    |
| ------------------------------------------- | ------------------------------------------------------------------------------ |
| `POST /domains`                             | Register a domain and return the DNS records you need to publish.              |
| `GET /domains`                              | List all domains on the team. Cursor-paginated.                                |
| `GET /domains/{public_id}`                  | Fetch a single domain's status and DNS records.                                |
| `POST /domains/{public_id}/verify`          | Trigger an on-demand verification check.                                       |
| `DELETE /domains/{public_id}`               | Remove a domain. Blocked while mailboxes still live on it.                     |

## Permissions

Domain endpoints are gated by the following API-key permissions. Keys with the wildcard `domain.*` satisfy all of them, and the built-in `root:full` role includes `domain.*`.

| Endpoint                              | Required permission |
| ------------------------------------- | ------------------- |
| `POST /domains`                       | `domain.create`     |
| `GET /domains`                        | `domain.read`       |
| `GET /domains/{public_id}`            | `domain.read`       |
| `POST /domains/{public_id}/verify`    | `domain.verify`     |
| `DELETE /domains/{public_id}`         | `domain.delete`     |

<Note>
  Keys provisioned with `root:readonly` carry `domain.read` so monitoring integrations can list your domains without being able to modify them.
</Note>
